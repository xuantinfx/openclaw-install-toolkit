Sharing one Mac with someone else? Read this.
==============================================

Who this is for
---------------

Two or more people sharing the same Mac, where each person has their
own macOS login (their own name in the top-right menu). If it's just
you on the Mac, ignore this file — follow instruction.txt instead.


What you each need (cannot be shared)
-------------------------------------

  - Your own Telegram bot.
      Each person runs BotFather and creates a bot under their own
      Telegram account. See instruction.txt, section A, for how.

  - Your own Anthropic API key.
      Each person signs up at console.anthropic.com and creates their
      own key. See instruction.txt, section B.

  - Your own copy of the toolkit zip, unzipped inside your own user
    folder (for example, your Downloads folder while logged in as you).


Why separate bots and keys
--------------------------

Only one bot can be "connected" to a given OpenClaw install. If you
and your housemate shared one bot, your messages would all land on
whichever Mac login happened to start first — and the other person's
Telegram would stay silent.

The Anthropic key is also tied to billing. Keeping keys separate means
each person pays only for their own usage.


How to install (each person, one at a time)
-------------------------------------------

  1. Log into your OWN macOS account. Not the other person's.

  2. Follow instruction.txt from STEP 1 through STEP 7. Every step
     works the same as if you were the only person on the Mac.

  3. Near the end of the installer, watch for a line like this:

         [OK] gateway healthy on 127.0.0.1:18790

     The number after the colon is YOUR port. The first person to
     install on the Mac usually gets 18789, the second person 18790,
     and so on. You don't need to type this number anywhere — just
     know it's yours.

  4. If you see a line like this during install, that's normal too:

         [port] 18789 is busy; using 18790 instead

     It means someone on this Mac already claimed 18789, so the
     installer picked the next free port for you automatically.


Can we install at the same time?
--------------------------------

Technically yes — but it's cleaner if one person finishes first. The
installer checks which ports are currently in use, so if two installers
run at the exact same moment, they might both see 18789 as free and
both try to use it. If that happens, whoever runs second will see their
bot fail to respond. Running the installer once more fixes it.


Troubleshooting
---------------

"no free port in 18789-18798"
  All ten reserved ports are taken. This usually means old OpenClaw
  daemons from past installs are still running in the background. Ask
  whoever set up the Mac to investigate, or run this in Terminal to
  see what's holding them:

      lsof -iTCP:18789-18798 -sTCP:LISTEN

Your bot replies with someone else's messages
  You're probably pointing at a bot that's already paired with another
  person's install. Open BotFather, create a NEW bot, and re-run the
  installer — it will ask for the new bot token.

Bot stopped replying after the other person installed
  Your own gateway service may have lost its port. Re-run install.sh
  from inside your toolkit folder. Answer "N" (or press Enter) when
  asked whether to wipe existing data — the installer will keep your
  config and just refresh the gateway service on a fresh free port.


Keeping things tidy
-------------------

Nothing crosses between accounts. Each person's bot token, API key,
config, and chat history live under their own macOS account (in
~/.openclaw). You cannot see each other's data unless someone shares
their password — keep it that way.
